Configuration Options

Choosing the right server for your business

When you run the configure command /pass configure you have three options to setup your server, they are described below:

Test server

This server is thinking for create and retrive dummy passwords in order to try the pass command workflow without the concerns of have to delete all the dummy data after that, this storage server is not permanent and the server is cleanup weekly.

It is ideal for:

  • Play and familiarize with the pass command

To setup this server execute the command /pass configure on any slack group and select the option Use Testing Server when you are done with your testing you can run the configure command again and pick another server.

Secure Server

This this our own maintained password server, this is a free and easy to setup password server, this is thinking to be used for many teams but each secret storage is only accesible for the creator team with the same security than the private servers but in this case the private key is unknow for any party more than us.

It is ideal for:

  • Share wi-fi passwords and others office scope passoword between the team members
  • Share user and password for non critical services between the team members

To setup this server execute the command /pass configure on any slack group and select the option Use Secure Server if you want to upgrade in any moment to a Private Server you can run the configure command to setup it.

Private Server

You can deploy your own private server, doing this you will have the total control an resposability of your private key, this is the most secure way to store your passwords without have to trust in any party to encrypt and decrypt them.

It is ideal for:

  • All the Secure Server mentioned uses.
  • Share access credential for company servers
  • Share user and password for critical services between the team members

To deploy a private server follow the github guide or contact us to quote a configuration. When you have your server up you will need to run the command /pass configure <private-server-url> on any slack group to start using it.
